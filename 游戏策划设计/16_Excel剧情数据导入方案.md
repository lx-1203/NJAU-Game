
## 16. Excel 剧情数据导入方案

> Excel Story Data Import Solution
>
> 本章回答三个核心问题：**能不能做？怎么做？需要什么工具？**
> 纯设计功能说明，不涉及代码实现。

---

### 16.1 结论：完全可以实现，且强烈推荐

当前 Unity 游戏系统**完全可以实现**通过 Excel 快速导入剧情。这是业界（尤其是文字冒险、模拟经营、Galgame 类游戏）的**标准做法**，不是什么黑科技。

**《钟山下》为什么特别适合用 Excel 管理剧情？**

| 问题                                   | 如果不用 Excel                        | 用了 Excel 之后                         |
| -------------------------------------- | ------------------------------------- | --------------------------------------- |
| 40 回合 × 每回合多个事件 = 上百条事件 | 在 Unity 编辑器里一条条手填，极其痛苦 | 在表格里批量编辑，一目了然              |
| 41 个结局，每个都有触发条件+描述文本   | 分散在代码各处，改一个找半天          | 一个 Sheet 里全部管理，改完重新导入即可 |
| NPC 对话、好感度、分支选项             | 在代码里写死，每次改都要重新编译      | Excel 里改完文字，游戏里直接生效        |
| 多人协作（你写主线，别人写支线）       | 互相改代码容易冲突                    | 各自负责不同的 Sheet，互不干扰          |
| 后期扩充大二~大四内容                  | 要反复改代码结构                      | 只需在 Excel 里追加新行                 |

---

### 16.2 整体思路：数据与逻辑分离

**核心理念：** 把「剧情内容」和「游戏程序」彻底分开。

- **剧情内容**（谁说了什么话、选了什么选项、产生什么效果）→ 放在 **Excel 表格**里管理
- **游戏程序**（怎么读取数据、怎么显示对话框、怎么计算属性）→ 放在 **Unity 代码**里实现

这样做的好处是：**改剧情永远不需要动代码，改代码永远不会丢剧情。**

```
┌──────────────────────────────────────────────────────────────────────┐
│                        日常工作流程                                  │
│                                                                      │
│   ① 策划/编剧在 Excel 里写剧情                                      │
│          ↓                                                           │
│   ② Excel 文件「另存为」或通过小工具，导出成游戏能读的格式           │
│          ↓                                                           │
│   ③ 把导出的文件丢进 Unity 项目的资源文件夹                         │
│          ↓                                                           │
│   ④ 运行游戏 → 剧情自动加载，立刻看到效果                          │
│                                                                      │
│   想改剧情？回到第①步，改 Excel，重复②③④，全程不碰代码             │
└──────────────────────────────────────────────────────────────────────┘
```

### 16.3 三种实现方案对比

|                              | 方案 A                         | 方案 B                       | 方案 C                     |
| ---------------------------- | ------------------------------ | ---------------------------- | -------------------------- |
| **名称**               | Excel → CSV → Unity          | Excel → JSON → Unity       | Unity 直接读 .xlsx         |
| **操作方式**           | Excel 里点「另存为 CSV」       | 用小工具自动转换             | 游戏直接打开 Excel 文件    |
| **上手难度**           | ★☆☆ 最简单                  | ★★☆ 需装一个小工具        | ★★★ 需装第三方插件      |
| **成本**               | 完全免费                       | 完全免费                     | 完全免费                   |
| **能处理的数据复杂度** | 简单表格（一行一条数据）       | 复杂结构（对话树、嵌套分支） | 同方案 B                   |
| **多 Sheet 支持**      | ❌ 每个 Sheet 要单独导出       | ✅ 一次性全部导出            | ✅ 直接读取                |
| **适合的数据类型**     | 属性数值表、物品表、简单事件表 | 对话系统、剧情分支、事件链   | 同方案 B                   |
| **缺点**               | 文本里有逗号/换行时容易出错    | 需要额外跑一步转换           | 游戏包体积变大，加载速度慢 |

**《钟山下》推荐：方案 B（Excel → JSON → Unity）**

理由：本游戏的剧情数据包含大量**嵌套结构**——比如一个事件里有3个选项，每个选项又有不同的属性影响和后续事件。这种「表格里套表格」的数据，JSON 格式天然支持，CSV 则力不从心。

> **简单数据**（如食物价格表、属性数值表）可以用方案 A 作为补充，两种方案可以共存。

### 16.6 表格之间的关联关系

```
┌──────────────┐
│  NPC信息表   │ ←── NPC 的基础设定（名字、性格、地点）
│  Sheet 8     │
└──────┬───────┘
       │ NPC名称
       ↓
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  NPC对话表   │     │  主线事件表  │     │  条件事件表  │
│  Sheet 2     │     │  Sheet 1     │     │  Sheet 3     │
└──────┬───────┘     └──────┬───────┘     └──────┬───────┘
       │                    │                    │
       │ 事件ID互相引用     │ 前置事件ID         │ 触发条件
       ↓                    ↓                    ↓
┌─────────────────────────────────────────────────────────┐
│                    游戏运行时                            │
│            按回合顺序 + 触发条件，加载对应事件           │
└────────────────────────┬────────────────────────────────┘
                         │
                         ↓ 40回合结束
                ┌──────────────────┐
                │  结局表 + 恋爱表  │
                │  Sheet 5 + 6     │
                │  按属性条件判定  │
                └──────────────────┘
```

> **关键设计：通过事件ID互相引用。** 比如主线事件 ME005 的「选项1后续事件」填写 D003（一段NPC对话），就实现了「主线事件触发NPC对话」的效果。所有 Sheet 通过 ID 串联，形成完整的剧情网络。

### 16.7 数据导入的完整流程设计

#### 阶段一：策划填表（持续进行）

| 步骤 | 具体操作                              | 说明                                   |
| ---- | ------------------------------------- | -------------------------------------- |
| 1    | 打开 `钟山下_剧情数据.xlsx`         | 按照 §16.5 的模板格式                 |
| 2    | 在对应 Sheet 里填写/修改剧情          | 例如在「主线事件表」里新增回合二的事件 |
| 3    | 用 Excel 的**数据验证**功能检查 | 确保事件ID不重复、引用的ID都存在       |
| 4    | 保存 Excel 文件                       | 建议同时用 Git 管理版本                |

#### 阶段二：格式转换（每次改完执行一次）

| 方式                           | 操作                                                                             | 耗时                |
| ------------------------------ | -------------------------------------------------------------------------------- | ------------------- |
| **方式 A（手动）**       | Excel 里选中一个 Sheet → 另存为 CSV 文件 → 放入 Unity 项目                     | 每个 Sheet 约 30 秒 |
| **方式 B（推荐：自动）** | 双击运行转换脚本 → 自动把所有 Sheet 一次性转为 JSON 文件 → 自动放入 Unity 项目 | 一键完成，约 2 秒   |

#### 阶段三：游戏加载（自动完成）

| 步骤         | 说明                                                                   |
| ------------ | ---------------------------------------------------------------------- |
| 游戏启动时   | 自动从资源文件夹读取转换好的数据文件                                   |
| 进入回合时   | 根据当前回合号，从主线事件表中找到对应事件并显示                       |
| 玩家做选择时 | 根据选项的「效果」字段修改对应属性，根据「后续事件」字段跳转下一个事件 |
| 游戏结束时   | 根据玩家最终属性，从结局表中按判定层级逐条检查，命中即停               |

### 16.8 Excel 表格编写规范

#### 命名规范

| 数据类型 | ID 前缀 | 示例           |
| -------- | ------- | -------------- |
| 主线事件 | ME      | ME001, ME002   |
| 条件事件 | CE      | CE001, CE002   |
| 固定事件 | FE      | FE001, FE002   |
| 特殊事件 | SE      | SE001, SE002   |
| NPC 对话 | D       | D001, D002     |
| NPC 信息 | NPC     | NPC001, NPC002 |
| 课程     | COURSE  | COURSE001      |
| 毕业结局 | END     | END001         |
| 恋爱结局 | LOVE    | LOVE001        |

#### 填写注意事项

| 规则                       | 说明                                                     |
| -------------------------- | -------------------------------------------------------- |
| **事件ID 必须唯一**  | 同一种类型内不能重复                                     |
| **效果格式统一**     | 用「属性名+数值」，多个用逗号分隔，如 `学力+5, 心情-3` |
| **空选项用 `-`**   | 没有第三个选项时不要留空白                               |
| **引用ID必须存在**   | 后续事件填的ID要在表里能找到                             |
| **描述文本可以换行** | 长文本在 Excel 单元格内换行即可                          |
| **属性效果可叠加**   | 一个选项可以同时影响多个属性                             |

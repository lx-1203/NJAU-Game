## 18. 考试系统

> Exam System
> 完整技术细节请参阅 `specs/exam-system-design.md`。
> **核心理念：** 玩家选了什么课，期末就考什么课。每门课考 3 道选择题，题目很简单（高中不及格都能答对），
> 重点是**沉浸感**——让玩家真正"坐在考场里"。体测采用独立的策略选择机制（见 §13）。

### 18.1 哪些事件触发选择题

**原则：一切跟"考试/测验"有关的事件，都让玩家做 3 道题。**

| 事件                       | 触发时机             | 考什么题           | 基础通过率                  |
| -------------------------- | -------------------- | ------------------ | --------------------------- |
| **期末考试(每门课)** | 每学期最后回合       | 对应课程题目       | 50% + 自习/出勤/突击修正    |
| **期中考试**         | 学期中间回合         | 该学期学分最高的课 | 70%（期中较容易）           |
| **四级考试**         | 对应回合             | 英语四级题目       | 30% + 背单词次数×5%        |
| **六级考试**         | 通过四级后           | 英语六级题目       | 20% + 背单词次数×5%        |
| **计算机等级考试**   | 指定回合             | 计算机题目         | 40% + 自习计算机课次数×10% |
| **补考**             | 挂科后下学期第一回合 | 挂掉那门课的题目   | 60%（补考题更简单）         |

#### 期末考试顺序

按课表从上到下依次考，每门课考 3 道题：

```
期末考试周流程：
  📝 第1场：[课程1] → 3道选择题
       ↓ 考间休息（室友对话过场）
📝 第2场：[课程2] → 3道选择题
       ↓ ...
  📝 继续直到所有需考试的课程考完
       ↓
  🏃 体育：体测（独立系统，见§13）
       ↓
  📊 全部考完 → GPA结算 → 学期总结
```

### 18.2 选择题设计原则

| 难度标准       | 说明                                      |
| -------------- | ----------------------------------------- |
| 正常玩家       | 3 道全对概率**≥ 85%**                    |
| 知识薄弱的玩家 | 3 道全对概率**≥ 60%**                    |
| 完全不懂的人   | 蒙对概率**≥ 25%**（4选1）                |
| 整体定位       | **初中~高一水平**，绝不超过高考难度 |

**四条设计原则：**

1. **一眼能看出正确答案** — 凭直觉/常识就能选对
2. **错误选项要有趣、有梗** — 让玩家即使知道正确答案也想看看其他选项
3. **融入校园生活场景** — 题目背景与大学生活相关
4. **正确答案位置随机** — A/B/C/D 随机分配

### 18.3 答题对通过率的影响

| 答题结果  | 通过率变化           | 附加效果         |
| --------- | -------------------- | ---------------- |
| 答对      | **+5%**        | 无               |
| 答错      | **-10%**       | 压力 +2          |
| 全对(3/3) | 额外**+5%** (共+20%) | 心情 +3          |
| 全错(0/3) | 无额外惩罚(已-30%)   | 压力 +5，心情 -5 |

### 18.4 考试完整流程（UX）

```
阶段一：考试周开始 → 展示本学期所有考试科目列表
阶段二：单科考前 → 展示该科的出勤、自习次数、突击复习等加成 → 计算基础通过率
阶段三：答题（不限时）→ 每题4选1，角落有半透明[👀偷看旁边的]按钮
阶段四：答题反馈 → 答对 ✅ +5%；答错 ❌ -10%
阶段五：单科结算 → 最终通过率 → 掷骰判定 → 通过/挂科 → 映射成绩(60~100分)→GPA
阶段六：考间休息 → 室友对话过场（叙事）
阶段七：全部考完 → 成绩单 → 学期GPA + 累计GPA
```

### 18.5 作弊系统

每道题界面角落有半透明按钮 `[👀 偷看旁边的]`：

| 项目            | 效果                                                      |
| --------------- | --------------------------------------------------------- |
| 点击后          | 直接告诉你正确答案，该题自动+5%                           |
| 被抓概率        | **30%**（每次独立判定）                             |
| 未被抓          | 黑暗值 +5，负罪感 +8                                      |
| 被抓后果        | **该科直接 0 分**，黑暗值 +10，负罪感 +15，压力 +20 |
| 累计被抓 ≥ 2次 | 触发强制结局「学术不端·开除」                            |

> 题目本身很简单，几乎不需要作弊就能答对。**选择作弊纯粹是一个道德选择**，与负罪感系统深度绑定。

### 18.6 最终通过率公式

```
最终通过率 = 基础通过率 + 答题修正 + 属性修正

基础通过率 = 50%
           + 自习该科次数 × 10%
           + 全勤 ? 15% : 0
           - 旷课>3次 ? 20% : 0
           + 突击复习 ? 20% : 0

答题修正 = 每题答对+5%, 答错-10%
         + 全对奖励+5%

属性修正 = -压力×0.1 + 幸运×0.1 - 负罪感×0.08

最终通过率 = clamp(最终通过率, 5%, 99%)
```

**典型场景：**

| 场景      | 基础 | 答题(预期全对) | 属性 | 最终 | 结果     |
| --------- | ---- | -------------- | ---- | ---- | -------- |
| 学霸      | 100% | +20%           | +3%  | 99%  | 稳过     |
| 普通学生  | 60%  | +20%           | +1%  | 81%  | 大概率过 |
| 摸鱼学生  | 30%  | +20%           | -3%  | 47%  | 看运气   |
| 摆烂+答错 | 30%  | -15%           | -5%  | 10%  | 基本挂   |

### 18.7 题库规模规划

| 题库类型       | 组数  | 题数       |
| -------------- | ----- | ---------- |
| 每门学科课程   | 6~8组 | 18~24题/课 |
| 四级专用       | 6组   | 18题       |
| 六级专用       | 6组   | 18题       |
| 计算机等级专用 | 6组   | 18题       |

> **"组"的概念：** 每次考试随机抽 1 组(3道题)，保证3道题之间有逻辑连贯性（从简到稍难），也方便后期扩充。

### 18.8 与其他系统的联动

| 系统                  | 联动方式                                   |
| --------------------- | ------------------------------------------ |
| 自习(固定事件#3)      | 自习选择课程 → 对应课程通过率+10%         |
| 上课/旷课(固定事件#1) | 每门课独立统计出勤 → 影响通过率           |
| 背单词(固定事件#7)    | 累计次数 → 影响四六级基础通过率           |
| 期末突击(条件事件#5)  | 消耗3行动点 → 所有科目通过率+20%          |
| 校园跑                | 累计次数影响体魄值 → 影响体测成绩         |
| 挂科(条件事件#4)      | 通过率<随机阈值 → 挂科 → 补考标记        |
| GPA系统(§11)         | 每门课成绩 → 绩点 → 加权平均 → 影响结局 |
| 负罪感/黑暗值         | 考试作弊 → 负罪感+8/+15，黑暗值+5/+10     |
| 金钱系统              | 体测代测 → 花费50元                       |
| 结局判定              | GPA + 四六级通过 → 决定学术/职场/仕途结局 |
| 学术不端结局          | 考试作弊被抓≥2次 → 强制开除              |


## 8. 校园新闻系统

> Campus News System

### 8.1 系统概述

#### 定位

新闻系统是每回合开始时呈现给玩家的**信息展示界面**，以"校园报纸 + 手机论坛"的混合形式，在玩家进入行动阶段前提供：

- **主线剧情预告** — 让玩家对即将发生的事件有预期和期待
- **校园趣闻轶事** — 增加世界观沉浸感与生活气息
- **隐性攻略提示** — 暗示属性门槛、NPC 动态、隐藏事件的存在
- **动态反馈** — 反映玩家上回合行为产生的连锁反应

> **核心理念："让玩家觉得这个校园是活的。"**
>
> 新闻不是纯文字堆砌，而是一个微型叙事系统——每条新闻都是一个小故事，有的会跨多个回合连载，有的会因玩家选择而产生不同版本。

#### 在回合流程中的位置

```
回合开始
  │
  ▼
┌──────────────────────────────────┐
│  📰 校园新闻界面（本系统）        │  ← 玩家可翻阅，点击"开始新的一天"关闭
│  ┌────────────────────────────┐  │
│  │ 头条新闻（主线相关）        │  │
│  ├────────────────────────────┤  │
│  │ 校园热搜 TOP3              │  │
│  ├────────────────────────────┤  │
│  │ 趣闻/吐槽/广告（随机2~3条）│  │
│  ├────────────────────────────┤  │
│  │ [开始新的一天 →]            │  │
│  └────────────────────────────┘  │
└──────────────────────────────────┘
  │
  ▼
强制阶段（课表处理）
  │
  ▼
自由行动阶段
```

### 8.2 新闻分类

| 类型        | 标识         | 来源              | 每回合数量  | 可否跳过   |
| ----------- | ------------ | ----------------- | ----------- | ---------- |
| 📌 头条新闻 | `HEADLINE` | 主线剧情绑定      | 固定 1 条   | 否（必看） |
| 🔥 校园热搜 | `TRENDING` | 动态 + 固定混合   | 固定 3 条   | 是         |
| 💬 树洞吐槽 | `GOSSIP`   | 随机池 + 条件触发 | 随机 1~2 条 | 是         |
| 📢 通知公告 | `NOTICE`   | 条件触发          | 0~2 条      | 是         |
| 🛒 广告推广 | `AD`       | 随机池            | 随机 0~1 条 | 是         |

**各类型说明：**

- **📌 头条新闻：** 每回合唯一，对应主线剧情"预告片"，占据界面最上方、字号最大、可带配图。
- **🔥 校园热搜：** 模仿微博热搜，每条一行附热度值。组成：1 条动态（基于玩家行为）+ 2 条固定/半随机。
- **💬 树洞吐槽：** 模拟校园匿名论坛，匿名 ID + 内容 + 点赞数，语气口语化，偶尔暗示攻略。
- **📢 通知公告：** 学校/辅导员的正式通知，条件触发，提醒考试/活动/截止日期。
- **🛒 广告推广：** 校园商家/学生创业广告，轻松搞笑。

### 8.4 动态新闻系统

#### 8.4.1 动态热搜生成规则

动态热搜根据玩家上回合的行为和当前属性状态自动生成。每回合**热搜第 3 条**为动态热搜位。

| 条件                | 动态热搜内容                                  | 热度 |
| ------------------- | --------------------------------------------- | ---- |
| 上回合旷课 ≥ 2 次  | #大一新生旷课被点名# "某同学你在哪？"         | 🔥45 |
| 学力 ≥ 150         | #南农学霸养成记# "有人绩点已经快满了？"       | 🔥67 |
| 魅力 ≥ 120         | #校园风云人物# "有个大一新生好像很受欢迎"     | 🔥82 |
| 体魄 ≥ 130         | #晨跑达人# "每天操场都能看到同一个人在跑步"   | 🔥38 |
| 压力 ≥ 70          | #大学生心理健康# "你最近还好吗？"             | 🔥55 |
| 心情 ≤ 30          | #关注大学生情绪# "如果你不开心，请看这篇"     | 🔥61 |
| 金钱 ≤ 500         | #月底吃土人# "离发生活费还有 X 天，撑住"      | 🔥73 |
| 金钱 ≥ 5000        | #南农土豪# "有人的生活费是我的三倍？？"       | 🔥48 |
| 某 NPC 好感度 ≥ 60 | #校园 CP 预警# "XXX 和某同学走得好近啊"       | 🔥95 |
| 负罪感 ≥ 50        | #校园不良风气# "听说有人在做一些不太正当的事" | 🔥41 |

[System.Serializable]
public class NewsItem
{
    public string id;               // 唯一ID，如 "R1_HEADLINE_01"
    public NewsType type;           // 新闻类型枚举
    public string title;            // 标题
    public string content;          // 正文内容
    public string author;           // 作者/来源（头条用）
    public string anonymousId;      // 匿名ID（树洞用），如 "匿名柠檬精"
    public int likes;               // 点赞数（树洞用）
    public float hotValue;          // 热度值（热搜用），单位：万
    public string hotTag;           // 热搜标签
    public int targetRound;         // 目标回合（0=任意回合可出现）
    public string seriesId;         // 连载ID（空=非连载）
    public int seriesOrder;         // 连载顺序
    public NewsCondition condition; // 触发条件（null=无条件）
    public NewsPriority priority;   // 优先级
    public Sprite illustration;    // 配图（可选，头条用）
}

public enum NewsType
{
    Headline,   // 头条
    Trending,   // 热搜
    Gossip,     // 树洞
    Notice,     // 通知
    Ad          // 广告
}

public enum NewsPriority
{
    Critical = 0,   // 必须显示（头条、重要通知）
    High = 1,       // 优先显示（动态热搜、条件树洞）

#### NewsCondition 触发条件

```csharp
[System.Serializable]
public class NewsCondition
{
    public ConditionType type;

    // 属性条件
    public string attributeName;    // 属性名："学力"/"魅力"/"体魄"等
    public CompareOp compareOp;     // 比较运算符
    public float threshold;         // 阈值

    // 行为条件
    public string actionId;         // 行为ID："skip_class"/"self_study"等
    public int actionCount;         // 行为次数

    // NPC条件
    public string npcId;            // NPC标识
    public int affectionThreshold;  // 好感度阈值

    // 状态条件
    public string stateFlag;        // 状态标记："is_depressed"/"is_class_leader"等
}

public enum ConditionType
{
    None,           // 无条件（固定新闻）
    Attribute,      // 属性达到阈值
    Action,         // 执行了特定行为
    NpcAffection,   // NPC好感度
    State,          // 特定状态标记
    Money,          // 金钱条件
    Round           // 特定回合
}

public enum CompareOp
{
    GreaterThan,
    LessThan,
    GreaterOrEqual,
    LessOrEqual,
    Equal
}
```

#### NewsManager 新闻管理器

```csharp
public class NewsManager : MonoBehaviour
{
    [Header("新闻数据")]
    [SerializeField] private NewsDatabase newsDatabase;

    private List<NewsItem> currentRoundNews = new List<NewsItem>();
    private HashSet<string> shownNewsIds = new HashSet<string>();

    /// <summary>
    /// 回合开始时调用，生成本回合新闻列表
    /// </summary>
    public List<NewsItem> GenerateNewsForRound(int roundNumber, PlayerData player)
    {
        currentRoundNews.Clear();
        AddHeadline(roundNumber);             // 1. 头条（固定，每回合1条）
        AddTrendingNews(roundNumber, player);  // 2. 热搜（2固定+1动态）
        AddGossipNews(roundNumber, player);    // 3. 树洞（随机1~2条+条件触发）
        AddNoticeNews(roundNumber, player);    // 4. 通知（条件触发）
        AddAdNews(roundNumber);                // 5. 广告（随机0~1条）
        AddSeriesNews(roundNumber);            // 6. 连载新闻
        return currentRoundNews;
    }

    private bool CheckCondition(NewsCondition condition, PlayerData player)
    {
        if (condition == null || condition.type == ConditionType.None)
            return true;
        switch (condition.type)
        {
            case ConditionType.Attribute:
                float attrValue = player.GetAttribute(condition.attributeName);
                return Compare(attrValue, condition.compareOp, condition.threshold);
            case ConditionType.Money:
                return Compare(player.money, condition.compareOp, condition.threshold);
            case ConditionType.NpcAffection:
                int affection = player.GetNpcAffection(condition.npcId);
                return Compare(affection, condition.compareOp, condition.affectionThreshold);
            case ConditionType.State:
                return player.HasStateFlag(condition.stateFlag);
            default:
                return true;
        }
    }
}
```

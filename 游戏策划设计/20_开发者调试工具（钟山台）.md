## 20. 开发者调试工具（钟山台）

> Developer Debug Tool — "ZhongshanConsole"

### 20.1 工具定位与核心理念

**工具名称：** 钟山台（取「钟山下」之意，「台」取「控制台」之意）

**一句话定位：** 一个仅在开发阶段可用的「上帝视角操控面板」，让开发者能够随意修改任何游戏数据、跳转到任意回合、强制触发任意事件，从而快速验证 41 个结局和所有数值公式的正确性。

**为什么必须有这个工具？**

| 问题                             | 没有调试工具时                                                        | 有调试工具时                                                                  |
| -------------------------------- | --------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| 验证 7★「校园传说」结局         | 需要正常玩完 40 个回合，每个回合小心翼翼地选择最优操作，耗时 2~3 小时 | 一键将所有属性设为 250+，GPA 设为 3.9+，负罪感设为 0，10 秒内直接触发结局判定 |
| 测试「网贷破产」强制结局         | 需要反复操作让金钱降到 -5000，中间还要躲过其他强制结局                | 直接将金钱设为 -4999，下一回合自动触发                                        |
| 检查负罪感 60→80 区间的分流行为 | 需要精确操控多回合的选择来把负罪感控制在这个区间                      | 拖动滑块到任意值，立即观察结局判定结果                                        |
| 验证「抑郁三级递进」             | 需要连续维持低心情 3+6+9 回合才能完整测试                             | 直接设置心情=20，快进 3 回合→检查轻度→再快进→检查中度→再快进→检查重度    |
| 测试所有 6 个恋爱结局            | 每个结局需要从头开始培养 NPC 好感度，6 次完整游戏流程                 | 直接设置恋爱状态、好感度、心情均值，6 个结局 2 分钟内全部验证                 |

**核心理念：**

- **不是玩家功能**——发布时完全隐藏/移除，不留任何入口
- **不是外挂**——不破坏游戏平衡，仅用于开发阶段验证数据正确性
- **像「上帝模式」**——开发者拥有对游戏世界的完全控制权
- **灵感来源**——类似《火山的女儿》开发时必然也有的内部测试工具、Unity 编辑器中的 Inspector 面板思路

### 20.3 呼出方式与界面设计

#### 呼出方式

| 方式     | 操作                        | 说明                              |
| -------- | --------------------------- | --------------------------------- |
| 快捷键   | 连按 `~`（波浪号）键 3 次 | 类似大多数游戏的控制台呼出方式    |
| 组合键   | `Ctrl + Shift + D`        | 备用呼出方式                      |
| 隐藏按钮 | 在设置界面连点版本号 5 次   | 类似 Android 开发者模式的呼出方式 |

#### 界面布局

```

│  🏔️ 钟山台 — 开发者调试工具                    [最小化] [关闭]  │
├─────────────────────────────────────────────────────────────────┤

│  [属性] [时间] [结局] [事件] [NPC] [经济] [公式] [快照] [日志]  │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│                      （各模块内容区域）                          │
│                                                                 │
│                                                                 │
│                                                                 │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  当前回合：第3回合（大一上·11月）  │  游戏状态：运行中           │
│  上次操作：学力 +50（手动修改）    │  快照数：3                  │
└─────────────────────────────────────────────────────────────────┘
```

**界面特性：**

- 可拖拽移动位置，不遮挡游戏画面
- 可最小化为屏幕角落的小悬浮球（显示当前回合号）
- 半透明背景，调试时仍能看到游戏画面
- 所有修改实时生效，游戏画面同步更新

| 操作           | 说明                                                       |

| 摆烂预设   | 摆烂值 115，学力 30，GPA 1.2                       | 测试「半途而废」边界 |
| 贫困预设   | 金钱 -4500，压力 70                                | 测试网贷破产触发链   |
| 随机生成   | 所有属性在合理范围内随机生成                       | 随机测试结局分布     |
| 恢复初始值 | 重置为游戏开局的默认值                             | 重新开始测试         |

**三种快进模式详解：**

| 模式     | 行为                                                                 | 适用场景                                 |
| -------- | -------------------------------------------------------------------- | ---------------------------------------- |
| 正常快进 | 完整执行每回合的所有事件、属性结算、条件判定，只是加速演出           | 想看完整流程但不想等动画                 |
| 跳过事件 | 执行属性结算和条件判定，但跳过事件的演出/对话/选择（自动按预设选择） | 想看数值变化但不看剧情                   |
| 纯跳转   | 不做任何结算，直接将回合数设为目标值，属性保持当前状态不变           | 只想到某个时间点后手动修改属性再测试结局 |

**「设为刚好满足条件」按钮的作用：**

点击后，自动将相关属性调整到**刚好达到触发阈值**，然后在下一回合正常触发该条件事件。例如：

- 点击抑郁事件的「设为刚好满足条件」→ 自动将心情设为 29，连续低心情回合数设为 2，下一回合进入第 3 回合即触发轻度抑郁。

这比手动调属性更方便，因为条件事件的触发条件可能涉及多个属性的组合。

### 20.6 发布时的处理方式

**核心原则：正式发布的游戏中，钟山台必须完全不可见、不可访问。**

| 处理方式   | 说明                                                        |
| ---------- | ----------------------------------------------------------- |
| 开关控制   | 通过一个全局开关控制整个调试工具的启用/禁用，正式发布时关闭 |
| 快捷键失效 | 发布版本中，所有呼出方式（快捷键、隐藏按钮）全部失效        |
| 界面不加载 | 发布版本中，调试工具的界面根本不会被加载到内存中            |
| 不影响性能 | 发布版本中，日志追踪等后台记录功能全部停止，不消耗任何性能  |

### 20.7 开发优先级

> 钟山台不需要一次性全部开发完成，应该随着游戏核心系统的推进逐步实现。

| 优先级       | 模块          | 依赖               | 建议时机                                 |
| ------------ | ------------- | ------------------ | ---------------------------------------- |
| **P0** | 📊 属性面板   | 角色属性系统完成后 | 最先开发，因为几乎所有测试都需要修改属性 |
| **P0** | ⏩ 时间控制   | 回合系统完成后     | 与属性面板同期，快进功能对测试至关重要   |
| **P1** | 💬 事件控制台 | 事件系统完成后     | 事件系统完成后立即开发                   |
| **P1** | 📋 日志追踪器 | 任何时候           | 越早越好，能帮助追踪早期 Bug             |
| **P2** | 🎯 结局模拟器 | 结局判定系统完成后 | 结局系统完成后必须立即开发               |
| **P2** | 👥 NPC 控制台 | NPC 系统完成后     | NPC 系统完成后开发                       |
| **P2** | 💰 经济控制台 | 金钱系统完成后     | 金钱系统完成后开发                       |
| **P3** | 🧮 公式验证器 | 各公式实现后       | 数值调优阶段                             |
| **P3** | 📸 快照系统   | 存档系统完成后     | 基于存档系统扩展                         |
| **P4** | 🎲 随机模拟   | 所有系统完成后     | 最终平衡性测试阶段                       |

### 20.8 与《火山的女儿》对标参考

> 《火山的女儿》作为本游戏的灵感来源，其开发过程中也必然需要类似的调试工具。以下是合理推测的对照表：

| 调试需求   | 火山的女儿             | 钟山下                   |
| ---------- | ---------------------- | ------------------------ |
| 属性修改   | 修改智力/武力/魅力等   | 修改学力/体魄/魅力等     |
| 结局测试   | 48+ 个结局需要逐一验证 | 41 个结局需要逐一验证    |
| 好感度调试 | 调整各攻略对象好感度   | 调整 NPC 好感度          |
| 时间跳转   | 跳转到女儿的不同年龄   | 跳转到不同学年/学期      |
| 数值平衡   | 检查养成属性增长曲线   | 检查学力/负罪感增长曲线  |
| 分支验证   | 验证分支对话的正确性   | 验证事件选项的效果正确性 |

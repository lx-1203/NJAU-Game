## 19. 恋爱系统

> Romance System
> 完整技术细节请参阅 `specs/romance-system.md`。
> **核心理念：** 恋爱不是"刷好感度的数值游戏"，而是"在有限行动点中做出取舍的情感叙事"。

### 19.1 系统架构

```
┌──────────────────────────────────────────────────────┐
│                    恋爱系统总控                        │
├──────────┬──────────┬──────────┬─────────┬────────────┤
│ 好感度   │ 状态机   │ 社交互动 │ 约会    │ 事件引擎   │
│ 模块     │ 模块     │ 模块     │ 模块    │ 模块       │
├──────────┴──────────┴──────────┴─────────┴────────────┤
│                   NPC 个性化档案                       │
├───────────────────────────────────────────────────────┤
│              恋爱结局判定器（毕业时触发）               │
└───────────────────────────────────────────────────────┘
```

### 19.2 核心设计原则

| 原则     | 说明                                           |
| -------- | ---------------------------------------------- |
| 资源竞争 | 恋爱消耗行动点和金钱，与学业、健康构成三角博弈 |
| 性格匹配 | 不同 NPC 喜好不同，没有"万能攻略路线"          |
| 双向关系 | NPC 有主动行为和情绪波动                       |
| 后果真实 | 告白失败、分手、劈腿都有持久影响               |
| 叙事驱动 | 每个关键节点都有专属文本                       |

### 19.4 数据结构

```csharp
[System.Serializable]
public class NPCRelationshipData
{
    public string npcId;
    public int affection;                 // 当前好感度 (0~100)
    public RelationLevel level;           // 关系等级枚举
    public RomanceState romanceState;     // 恋爱状态枚举
    public int romanceHealthScore;        // 恋爱健康度 (0~100)
    public int romanceDurationRounds;     // 恋爱持续回合数
    public float romanceMoodSum;          // 恋爱期间心情总和
    public int consecutiveNoInteract;     // 连续未互动回合数
    public int confessionFailCount;       // 告白失败次数
    public int cooldownRoundsLeft;        // 冷却剩余回合
    public int breakupCount;              // 分手次数
    public bool isPermanentlyBroken;      // 是否永久决裂
    public int sameActionRepeatCount;     // 同一行动连续执行计数
    public string lastActionType;         // 上次互动类型
    public int nextAnniversaryRound;      // 下次纪念日回合
    public List<MemoryEntry> memories;    // 恋爱回忆录条目
}

public enum RelationLevel
{
    Stranger,       // 陌生人 0~19
    Acquaintance,   // 点头之交 20~39
    Friend,         // 朋友 40~59
    CloseFriend,    // 好友 60~79
    BestFriend,     // 挚友 80~100（未恋爱）
    Lover           // 恋人 80~100（已恋爱）
}

public enum RomanceState
{
    None,           // 无
    Crushing,       // 暧昧期（好感60~79）
    Cooldown,       // 告白失败冷却期
    Dating,         // 恋爱中
    BrokenUp,       // 已分手
    Hostile         // 敌对（因劈腿等）
}

[System.Serializable]
public class MemoryEntry
{
    public int round;
    public string semester;
    public string eventName;
    public string description;
    public bool isKeyMoment;
}
```

### 19.5 MVP 裁剪方案

**MVP（大一上·5回合）保留：**

- 好感度基础系统（5级 + 4种基础互动）
- NPC 性格偏好
- 好感度自然衰减 + 重复行动衰减
- 暧昧期随机事件（3个）

**MVP 砍掉：** 恋爱维护/健康度、纪念日、分手/复合、劈腿、天气约会修正、恋爱回忆录、异地恋判定（均需长期数据，5回合太短）
